A tactical AI microservice that listens to J-chat messages from IRC, assesses them using a local LLM, and returns enriched battle JSON to an orchestrator. Also accepts reassessment triggers from the orchestrator cluster via SSE.

---

## What It Does

PAE has two input paths that feed the same AI pipeline:

**Path 1 вҖ” IRC (new J-chat messages)**
```
IRC J-chat вҶ’ filter вҶ’ AI assessment вҶ’ POST /paeoutputs to orchestrator
```

**Path 2 вҖ” SSE (reassessment requests from the cluster)**
```
Orchestrator SSE event вҶ’ AI reassessment вҶ’ POST /paeoutputs to orchestrator
```

For every valid message, the AI assigns three prioritised effect operator verbs (e.g. `STRIKE`, `DETECT`, `JAM`) and populates a full battle JSON record with tactical justifications, time windows, entities of interest, and operational limits. Each assessment is simultaneously pushed to the live dashboard UI.

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env` with your values:

```env
# IRC
IRC_SERVER=10.5.185.72
IRC_PORT=6667
IRC_CHANNEL=#app_dev

# LM Studio (running on your network)
LM_STUDIO_URL=http://10.5.185.55:4334/v1/chat/completions
LM_MODEL=google/gemma-4-e4b
LM_TIMEOUT=20

# Orchestrator
ORCHESTRATOR_BASE_URL=http://10.5.185.XX:PORT
ORCHESTRATOR_API_KEY=your-api-key-here

# SSE
SSE_RETRY_DELAY=5

# Database
DB_HOST=10.5.185.53
DB_NAME=shooca_db
DB_USER=shooca
DB_PASSWORD=your-password
DB_PORT=5432
```

### 3. Run locally

```bash
python main.py
```

## Docker

Run the full stack as containers:

```bash
docker-compose up --build
```

This starts three services from a single shared Docker image:

| Service | Container | Port | Purpose |
|---|---|---|---|
| `emulator` | `pae_emulator` | `3016` | Fake orchestrator for testing |
| `pae` | `pae_app` | вҖ” | Main PAE application |
| `config` | `pae_config` | `8080` | Config editor + live dashboard UI |

> Comment out the `emulator` service in `docker-compose.yml` when connecting to the real orchestrator.

```env
LM_STUDIO_URL=http://host.docker.internal:4334/v1/chat/completions
```

## Web UI вҖ” Port 8080

When running (locally or in Docker), open `http://localhost:8080` in your browser. Two pages are available:

### Config вҖ” `http://localhost:8080/`

Live editor for PAE environment settings. Update the following without restarting the container:

- Orchestrator Base URL and API Key
- LM Studio URL and Model
- IRC Server and Channel
- SSE Retry Delay

Changes are written to `.env` immediately. The SSE client auto-reconnects so orchestrator URL changes take effect within seconds. Changes to `IRC_SERVER` or `LM_STUDIO_URL` require a container restart.

### Dashboard вҖ” `http://localhost:8080/dashboard`

Real-time view of every AI assessment as it happens.

**Live Feed** вҖ” top section shows the last 3 assessments auto-expanded as they arrive via SSE. Each card displays:
- Original J-chat message
- AI-generated label and strategic description
- Source (IRC or SSE) and originator
- All three battle effects with effect operator verbs, justifications, time windows, state hypotheses, and ops limits
- Entities of interest and battle entity tags

**History** вҖ” scrollable list of all assessments since the server started, collapsed by default with click-to-expand.

The dashboard reconnects automatically if the connection drops.

## Local Testing (without the real cluster)

The emulator fakes the orchestrator so you can test the full pipeline locally.

**Terminal 1 вҖ” start the emulator:**
```bash
python tests/emulator/pae_run_emulator.py
```

**Terminal 2 вҖ” update `.env` and start PAE:**
```env
ORCHESTRATOR_BASE_URL=http://127.0.0.1:3016
ORCHESTRATOR_API_KEY=test
```
```bash
python main.py
```

**Fire a reassessment trigger (PowerShell):**
```powershell
Invoke-WebRequest -Uri "http://127.0.0.1:3016/paeinputs" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"requestId":"test-001","trackId":"AMTI SAT detected TBM launch preparations at grid PB1.2","originator":"test-operator"}'
```

**Fire a reassessment trigger (curl / Linux / Mac):**
```bash
curl -X POST http://127.0.0.1:3016/paeinputs \
  -H "Content-Type: application/json" \
  -d '{"requestId":"test-001","trackId":"AMTI SAT detected TBM launch preparations at grid PB1.2","originator":"test-operator"}'
```

**Check stored assessments:**
```
http://127.0.0.1:3016/paeoutputs    вҶҗ orchestrator store
http://127.0.0.1:3016/docs          вҶҗ Swagger UI
http://localhost:8080/dashboard     вҶҗ live dashboard
```

## Output Schema

Every assessment produces a battle JSON record structured as:

```json
{
  "id": "<originator>",
  "requestId": "<unique track id>",
  "label": "<short tactical title>",
  "description": "<strategic summary>",
  "entitiesOfInterest": ["<locations, systems>"],
  "battleEntity": ["<vehicle or actor types>"],
  "battleEffects": [
    {
      "id": "pae-002-e01",
      "effectOperator": "<VERB>",
      "description": "<justification>",
      "timeWindow": "<urgency estimate>",
      "stateHypothesis": "<tactical outcome>",
      "opsLimits": [{ "description": "...", "battleEntity": "...", "stateHypothesis": "..." }],
      "goalContributions": [{ "battleGoal": "2.1.c", "effect": "high" }],
      "recommended": true,
      "ranking": 1
    }
  ],
  "chat": ["<original message>", "PAE generated for pre-emptive and defensive options."],
  "isDone": false,
  "originator": "<source>",
  "lastUpdated": "<ISO timestamp>"
}
```

## Key Behaviour Notes

- Messages shorter than 2 words or under 6 characters are filtered as noise
- The AI selects three verbs from an approved battle dictionary across four categories: `ATTACK`, `INVESTIGATE`, `DEGRADE`, `RESCUE`
- CSV reference tables are looked up per-message вҖ” only rows matching terms in the message are sent to the AI to keep the prompt lean
- If the AI returns nothing the record is marked `NO PAE ACTION REQUIRED`
- All assessments are written to `tactical_output.log` locally regardless of orchestrator availability
- The `opsLimits.battleEntity` field is always guaranteed to be populated вҖ” if the AI omits it a fallback value of `"Unspecified"` is injected before submission
- The dashboard stores up to 200 assessments in memory вҖ” history resets if the config container restarts
